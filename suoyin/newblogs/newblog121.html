【LODOP】安装旧版本导致注册号无效的一种情况（旧版本更新）（newblog121）（文案，初）

windows版的lodop中，最常见的是客户端本地打印角色，第一种角色：

三种角色及使用方法：
参考http://www.c-lodop.com/demolist/t1.html

第一种角色在需要去掉直接打印的水印时，可以通过ip或域名访问地址注册，也可以通过公司名称注册。

之前有视频介绍了一些ip或域名注册号无效的一些可能：
（图）
【LODOP】ip或域名注册号无效的一些排查https://v.ixigua.com/sEwq4vPwrhE/

此视频重点介绍版本问题的导致的注册无效问题。

有关版本的，之前视频里介绍了这两条，第4条和第5条：
4.很久以前的注册（例如11年之前的），使用了c-lodop
由于lodop很久以前只有插件方式，当时并没有推出c-lodop，c-lodop在当时属于未来功能。
原注册号只是针对lodop插件方式的。

5.安装版本太低，例如：安装的是lodop插件旧版本6198
（1）安装插件版本是6198，很早之前的客户，用的是只用插件方式，未升级过c-lodop，域名注册号无效
（2）注册语句书写正确，访问域名正确，执行正确，水印里带6198试用版字样
收注册号邮箱里会说明该注册号对多少版本之后的版本有效，如果出现问题，可安装最新版对比下。

注册号无效排查：
排查一下http://www.c-lodop.com/faq/ph01.html
该排查步骤第12条，有个关于版本号太低的排查步骤：
（图）
12.lodop版本太低，收注册号的邮箱里会有说明，例如：Lodop(6.111及之后版本)，排查版本号是否过低导致注册号无效，可安装最新版对比测试。

这里只是举例说明，具体是需要多少版本之后，具体还需要查看收注册邮箱里的说明。
并不是指6.111及之后的版本注册是有效的。

截止视频发布时间，手动的注册邮件里可能是 这样的说明
（图）
Lodop(6.192及之后版本)，收注册号邮箱里可能会显示这个版本，如果邮箱里显示的是这个，说明必须安装6.192及之后版，之前的版本是无效的。

6.111到6.191,，这之间的版本，以及6.111之前的版本都是无效的。

有个很久之前的项目，这个项目要更换域名了，重新购买了新域名的，注册无效是怎么回事？
既然是很久以前的，查看下当前安装的版本，是否是收注册邮箱里的版本号之后的版本。
如果是收注册号邮箱里的版本号之前的版本，是无效的。

解决方法是：
安装最新版，http://www.c-lodop.com/download.html

如何提示客户端下载安装最新版？

lodopFuncs.js里写好了判断提示安装和升级，可放置安装文件在自己项目里，
修改lodopFuncs.js里的提示下载路径换成自己项目的路径，该lodopFuncs.js在下载中心发行包里
http://www.c-lodop.com/download.html
（图）

修改提示升级判断版本，可提示客户端下载安装最新版：
LODOP.VERSION < "6.2.2.6"
（图）
当安装版本小于判断版本时，会提示插件方式的升级。

lodopFuncs.js是混合部署的，如果需要只用lodop插件方式，需要修改needclodop返回false,也就是都走插件方式。

由于当前大部分浏览器都不支持插件方式里，所以lodop已停止了更新，目前的更新都是针对c-lodop了，
Lodop控件升级到C-Lodop云打印 参考http://www.c-lodop.com/faq/pp7.html
//====判断是否需要安装CLodop云打印服务器:====
function needCLodop(){
try{
return true;
} catch(err) {return true;};
};
在这个链接里有修改称单独c-lodop方式的，如果是旧项目，只用的插件方式的，也可以改成false，只使用lodop插件。

需要修改的地方:
该lodopFuncs.js修改成只用lodop插件方式，提示升级版本为6.2.2.6，提示升级的安装文件路路径改成自己项目的路径，放置最新版插件6.2.2.6的安装程序。

这样，就可以提示所有客户端升级成6.2.2.6版，6.2.2.6版是6.192之后的版本，新版的注册号就可以生效了。

（---注意：高版本的火狐谷歌不再支持np插件，这种单独插件方式的，只适用于IE浏览器和一些壳浏览器的兼容模式--）
这种比较旧的项目，一般会有要求客户使用IE或一些壳浏览器的兼容模式，IE内核会走插件方式，如果能限制客户不用谷歌或一些壳浏览器的极速模式，也可以不修改这个needCLodop返回false，这个needCLodop就是根据浏览器内核判断使用哪个的。

随着IE浏览器和IE内核的浏览器较少了，如果可以，建议升级到c-lodop，跳出了浏览器的限制，可以支持所有浏览器。

购买了域名注册，但旧项目里安装插件版本太低，注册号无效，可以退款吗？
建议提示客户端升级到最新版插件，升级到收注册号邮箱里要求的版本号之后的版本。

域名不更换了，项目打算换回之前的旧项目的域名，已经购买了新版的域名注册号可以退款吗？
因注册号是虚拟商品，一经售出无法收回，为避免一些纠纷，建议通过升级插件版本方式让最新版注册号生效。

为什么注册价格表和商品详情里，未标明有版本限制？
收注册号邮箱里有标注XXX版本之后有效，关于版本无效问题，可以通过升级最新版插件解决。
虽然lodop插件方式不再更新，但目前下载中心发行包还保留了新版插件，是为了一些旧项目可以升级使用。

总结：
插件版本过旧导致注册无效怎么解决？
让客户升级插件版本，收注册号的邮箱里会有说明，例如：Lodop(XXXX及之后版本)，让客户升级到该版本之后的插件版本。

域名想换回之前的域名，新买的域名是否可以退款？
联系下淘宝旺旺客服，虚拟商品退款可能会有一些纠纷，说明情况。如果有必要，关于旧项目（10多年前很久以前）插件方式，具体就版本对新版注册号支持情况可以提前咨询下。
如在购买后发生旧版本不生效情况，建议让客户升级最新版插件，具体代码不用变动，只需要增加提示客户端升级的地方，提示所有客户端安装最新版插件。

这里简单回顾一下里面的以一些常见的可能错误，只介绍关于ip或域名注册的：
1.常见问题：
问题（1）.注册的是某域名，在本地测试的，没有发布到自己的域名上，浏览器地址栏不是注册的地址。
问题（2）.购买的是ip注册，实际是域名访问方式，误以为注册解析到该域名的ip可以用，浏览器地址栏不是注册的地址。
问题（3）.误认为是clodopFuncs.js的地址，不是注册的网站的访问地址
问题（4）.注册域名较长，购买时有中间字母错误，注册域名和实际使用访问的域名不一致
问题（5）.注册的域名填写时有空格，例如.com和之前的字母之间有空格，注册域名和实际使用访问的域名不一致

2.注册语句执行问题。
（1）注册语句放到lodoFuncs.js里，页面使用时没有在任务里调用getLodop，在打印方法外调用
（2）误认为调用一次注册一下就行了，调用一次后把注册语句注释掉了，没有每次打印时调用
（3）页面引用了包含注册语句的lodopFuncs.js，但页面通过c-lodop端口调用了clodopFuncs.js，代码里没走带注册语句的getLodop方法。
（4）页面引用了包含注册语句的lodopFuncs.js，页面调用的是getClodop，是clodopFuncs.js里的方法。
（5）页面引用了包含注册语句的lodopFuncs.js，页面还引用了obj的lodop插件对象，最后走的是插件方式，没走lodopFuncs.js里的getLodop方法。
（6）.......（其他各种执行原因，如调用多个lodop对象等...)
（7）浏览器换成问题，加上了注册语句，但缓存还是旧的JS，导致实际没后执行注册语句

3.注册语句书写错误
（1）不是复制的邮箱里的语句，误把字母O写成数字0，造成注册语句里的注册码是错的
（2）不是复制的邮箱里的语句，LODOP.SET_LICENSES的下划线误写为两个下划线
（5）语句里注册码错误，由于之前购买人离职，公司人员变动，人为故意留错注册码等造成。
（6）ip域名注册，却在语句里自己加了公司名参数
（7）注册语句加了空格或缺少空格，不是复制邮箱里的语句。
