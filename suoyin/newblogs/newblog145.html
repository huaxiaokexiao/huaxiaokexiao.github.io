【LODOP】谷歌147版本修改设置，CORS跨域报错（newblog145）（文案，初）

之前的视频介绍过一些谷歌浏览器更新导致的CORS报错的一些相关：
【LODOP】谷歌108后 http的clodopFuncs.js报CORS跨域报错 https://v.ixigua.com/b9PmyqQnV0c/
LODOP】CORS报错Web打印服务CLodop... https://v.douyin.com/gBGkXWEqGgU/
【LODOP】更新LodopFuncs.js谷歌跨域cors,142.. https://v.douyin.com/oAqjEKz9X_4/
（图）

当谷歌升级到147.0.7727.56（正式版本）后，会发现之前视频里介绍的一些解决方法已经不生效了。
先介绍下谷歌升级到147后，如何通过修改谷歌浏览器设置解决该问题：
操作步骤：
让客户在 Chrome 地址栏输入：chrome://flags/#local-network-access-check
找到 “Local Network Access check” 这个选项。
将其从 Default 改为 Disabled。
点击右下角的 “Relaunch” 重启浏览器。

让客户在 Chrome 地址栏输入：chrome://flags/#local-network-access-check
找到 “Local Network Access check” 这个选项。
（图）

将其从 Default 改为 Disabled。
点击右下角的 “Relaunch” 重启浏览器。

6636及之前版本：
原来可以通过：
安装最新版c-lodop，更新最新的lodopFuncs.js
这种方式就可以解决，最新版lodopFuncs.js在下载中心发行包里，增加了ws方式，之前是可以解决的
（图）
6.5.6.5
增加：新增WebSocket方式加载CLodopfuncs.js从而进一步解决CORS问题，特别是新版外壳浏览器问题;
安装6.5.6.5之后的版本，及最新的带ws的lodopFuncs.js可解决谷歌147之前的谷歌浏览器的cors跨域问题。

但谷歌谷歌升级到147.0.7727.56（正式版本），之前的这种方法已不再生效。
目前可用解决方法：
1.该视频最前面提到的，修改谷歌浏览器设置Local Network Access check将其从 Default 改为 Disabled，并重启浏览器。
2.换低版谷歌或360这种兼容浏览器（360浏览器，搜狗浏览器，QQ浏览器）
3.把网站地址配置成HTTPS页面
4.安装6636之后的版本，有可能后续会解决该问题（不包含6636）
（截止视频发布时间附近，最新版6638版）

谷歌147版本及后一律要求https通讯协议，不再支持http(非安全协议)访问本地设备。
将来谷歌升级152版，可能还会有问题，因此提前把http网站转成https能更好解决该问题，以及谷歌后续升级可能带来的问题。
