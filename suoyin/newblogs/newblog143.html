【LODOP】多页打印维护的调整（对象类名）维护一页，所有页都被维护（newblog143）（文案，初）

样例26演示了批量打印，还有批量打印维护模板的问题，这里详细解释和介绍下为什么要用类名，如何维护一页模板，该任务所有页的打印项都会被维护。
会介绍一些常见的问题和错误用法导致的现象。

首先介绍下什么是打印维护：
打印维护（PRINT_SETUP语句）功能是提供给客户端单个电脑微调用的，为了让微调结果永久有效，Lodop会在本地为该过程对应生成一个ini配置文件，文件路径和文件名类似如下： 系统盘符:\Program Files (x86)\MountTaiSoftware\Lodop\PrintSetups_打印任务名.ini

其中的“打印任务名”是初始化语句的参数设置的：
LODOP.PRINT_INIT("具体的打印任务名");
或： LODOP.PRINT_INITA(0,0,800,600,"具体的打印任务名");

最常见的就是解决不同打印机的偏移问题，有些打印机不在初始位置，可以根据该打印机用打印维护本地整体偏移，针对该打印机的内容偏移位置进行调整。
打印维护由于截图在客户端本地生成ini文件，所以对其他客户端没有影响。
当有打印维护本地配置的对应任务名的ini文件2时，不会走代码里的语句，会根据ini结果显示预览和打印。

上面是对打印维护的简单介绍，下面是演示对象类名，是如何维护一页，让该任务所有页的同类名的打印项都被维护的。
（图）

还是上面那个例子，一个只有两个text的简单页面：
由于需要加类名，ADD_PRINT_TEXT换成ADD_PRINT_TEXTA，第一个参数为类名
function CreateOnePage(W3,W4){
LODOP.NEWPAGEA();
LODOP.ADD_PRINT_TEXTA("text01",25,34,100,20,W3);
LODOP.ADD_PRINT_TEXTA("text02",61,34,100,20,W4);
};

在一个任务里，循环10页这样的页面：
function prn1_preview() {
LODOP=getLodop();
LODOP.PRINT_INIT("克里斯蒂");
for (j = 1; j <=10; j++) {
CreateOnePage("A文本"+j,"B文本"+j);
}; ;
LODOP.PREVIEW();
};

打印维护功能，只写一页的：
function prn1_setup() {
LODOP=getLodop();
LODOP.PRINT_INIT("克里斯蒂");
CreateOnePage("A文本1","B文本2");
LODOP.PRINT_SETUP();
};

点击打印维护，随便拖动第二个打印项到右边一些，点应用，可以看到当前打印维护只有两个打印项，也就是1一页。
（图）

再进入打印预览，切换后面的页数，可以看到不只是第一页的打印项，后面的所有同类名的打印项都被维护了，第二个text位置变为了打印维护后的位置。
（图）

为什么要用类名，不用类名不行吗？
不行，10页循环里有多个打印项，如果想维、护一页，该任务所有页都被维护，需要设置相同的类名，例如上面的例子，第二个text都有同样的类名：text02

这边也演示下不用类名会有什么现象？
（图）
LODOP.NEWPAGEA();
LODOP.ADD_PRINT_TEXT(25,34,100,20,W3);
LODOP.ADD_PRINT_TEXT(61,34,100,20,W4);

打印维护还是移动下，点应用，进入预览，可以看到第一页是维护后的结果，但第2页以及之后的页数并没有被维护，还是走的代码的里的位置。
（图）

如果想同一个模板，维护一页，该任务所有页都被维护，则可以用设置类名方式。
