在排查一些问题时，经常用到查看打印机队列，这里说明一些打印过慢的可能情况，和队列的介绍。

如何查看打印机队列？
windows系统中，当在打印时，右下角会出现一个小打印机图标，点击可以查看到当前打印的情况。

如果打印机和电脑线没连接脱机等，会报错，一般windows右下角也会有提示。

当打印完成后，右下角图标会消失。

如何在windows查看每个打印机当前的打印情况，如何打开队列？
在控制面板，找到打印机，点查看打印详情。

这个是win10的：

（图）
（图）
查看设备和打印机——选中需要查看的打印机——点上面的查看正在打印什么
这样就可以看到当前打印机的队列里的情况。

也可以选中打印机，在右键菜单里，查看正在打印什么。

观察打印机队列有什么用？
在批量打印时，可以观察每个任务加入队列的时间。
给任务名加序号或时间戳，还可以观察每个任务的情况，是否有重复打印或漏打等。

可以协助排查一下问题。

打印卡慢，可能和很多原因有关：打印内容或样式或图片等过多，有需要下载有脚步执行或本身网络慢；
机器性能过低 系统ie有问题或缓存过多；或使用的是共享打印机。
如果预览页数多，可以多分一些任务，太多不建议预览，建议直接打印，在任务和页数之间平衡一下，参考样例26
http://www.c-lodop.com/demolist/PrintSample26.html

一个任务分多少页合适呢？
需要根据自己的任务内容等实际测试下，可以观察下队列的任务加入的速度。

（测试时，可以打印机脱机，防止真的打印出来浪费纸张）
(图）


直接复制一下欢迎页面的例子，把htm超文本去掉，只保留最简单的一个纯文本，一个任务一页，查看下当前队列的情况
for (j = 1; j <=10; j++) {
LODOP.PRINT_INITA(1,1,770,660,"测试");
LODOP.SET_PRINTER_INDEX(iPrinterIndex);
LODOP.ADD_PRINT_TEXT(10,60,300,200,"这是测试的纯文本，下面是超文本:");
LODOP.PRINT();
};

(图）
可以看到，由于一个text文本内容比较简单，所以速度非常快。
队列里可以看到当前任务的页数，大小，和提交时间。

根据队列的情况，可以尝试调整分页分任务或分组打印，也可以排查当前页数等是否正常。
(图）

测试完成后别忘了取消所有文档，这个队列里排队是正在等待打印的状态，取消脱机等，会加入实际打印当中，测试完成点取消所有文档。
(图）

较大的任务，或内部有需要解析的内容过多等，会比较慢，两个任务之间间隔比较长，打印机打印时，可能就会感觉打了一个任务，要过一会才能打印下一个，造成卡顿现象。
前面任务已经打完了，后面的任务还没加入队列完成，容易造成这种现象。

较大的任务，页数较大，打印可能会存在比较慢的情况。
(图）

造成比较慢的具体原因，需要排查下打印内容，例如pdf打印项，有些pdf加载较慢，有些图片或样式过多等，或打印机驱动解析存在问题。
如果是该电脑存在某个共享打印机故障，也可能会导致一些异常。

之前视频里提到过一个整页旋转的问题：
（图）
如果有这个问题，也可以更换旋转方式。

不确定是哪个样式或语句引起的，怎么办？
可以逐个删减排查，或做简单例子逐步增加排查，定位到哪里引起的。

这里再修改下欢迎页面的例子，把欢迎页面的那个htm也加上，再查看下队列情况
for (j = 1; j <=10; j++) {
LODOP.PRINT_INITA(1,1,770,660,"测试预览功能");
LODOP. SET_PRINTER_INDEX("OKI MICROLINE 5200F+");
LODOP.ADD_PRINT_TEXT(10,60,300,200,"这是测试的纯文本，下面是超文本:");
LODOP.ADD_PRINT_HTM(30,5,"100%","80%",strHTML);
LODOP.PRINT();
};

对比可以看到，一个text打印非常快，加了一个htm后，任务之间的间隔时间就增加了。
（图）（图）

可以根据实际内容平衡一下任务数量和页数，由于htm需要解析，纯文本速度会更快，如果内容简单，也可以尝试纯文本输出方式。
快递单，小票，标签，票据等，建议用ADD_PRINT_TEXT等纯文本和画线等进行设计，不受浏览器解析的影响。
参考http://www.c-lodop.com/demolist/PrintSample4.html

批量打印较多较大的pdf可以吗？
理论上没有限制，但受硬件，系统等影响，pdf来源网络也受影响

如果量实在较大，也可以让客户下载pdf，客户本地自己打印。这样就不用通过网络批量在线打印大量pdf了。

批量打印有最大数量吗？
理论上没有限制，但受硬件等影响，操作系统windows服务过多也可能会打崩溃，打印机驱动所能承受的也有限制。
可以根据实际效果分任务分页分组打印。
